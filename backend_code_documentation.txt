

### `backend/check_session.php`

该文件是一个 API 端点，用于检查用户当前的登录状态。前端应用程序可以在加载时调用它，以确定是应该显示用户相关内容还是将用户重定向到登录页面。

- **会话依赖**:
    - 首先加载 `api_header.php`，以确保会话已正确启动和配置。

- **认证检查**:
    - 通过检查 `$_SESSION['user_id']` 是否存在来判断用户是否已经通过了身份验证。

- **JSON 响应**:
    - **如果已登录**: 返回一个 JSON 对象，其中 `isAuthenticated` 字段为 `true`，并附带存储在会话中的基本用户信息（如 `id` 和 `email`）。
    - **如果未登录**: 返回的 JSON 对象中 `isAuthenticated` 字段为 `false`。

这个脚本是实现前端路由保护和根据用户登录状态动态渲染UI（用户界面）的关键。
